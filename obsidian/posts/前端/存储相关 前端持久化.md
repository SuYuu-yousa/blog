# 前端持久化

## 基础概念

- 什么是前端持久化（为什么需要在客户端存储数据）

  **将数据存储在客户端（浏览器）中，使其在页面刷新、关闭甚至重启浏览器后依然能够保留和恢复** 的技术。

- 持久化 vs 临时存储的区别

  - 关了浏览器还在就是持久化，否则就是临时存储,特例：cookie过期，sessionstorage

  - *持久化的核心是* **数据写入磁盘***，临时存储的核心是* **数据只在内存中***。*

    

- 前端持久化的常见应用场景（用户偏好、离线缓存、表单草稿、Token 存储等）

## 二、各种存储方案详解

### 1. Cookie

- 基本原理与工作机制
- 属性详解（`Expires`、`Max-Age`、`Domain`、`Path`、`HttpOnly`、`Secure`、`SameSite`）
- 大小与数量限制
- 与服务端的交互（自动携带）
- 优缺点

### 2. Web Storage

- LocalStorage
- LocalStorage 只支持字符串，对象/数组必须 `JSON.stringify` 存，`JSON.parse` 取。
  - API 用法
  - 持久化特性（无过期时间）
  - 同源策略
  - 容量限制（约 5MB）
- SessionStorage
  - 与 LocalStorage 的区别（会话级别）
  - 标签页隔离机制

### 3. IndexedDB

- 基本概念（NoSQL、事务型数据库）
- 核心概念（Database、ObjectStore、Index、Transaction、Cursor）
- 原生 API 使用
- 常用封装库（Dexie.js、localForage、idb）
- 适用场景（大量结构化数据、离线应用）

### 4. Cache API / Service Worker 缓存

- Cache API 基本用法
- 与 Service Worker 配合实现离线缓存
- 缓存策略（Cache First、Network First、Stale While Revalidate 等）

### 5. 其他/补充方案

- WebSQL（已废弃，了解即可）
- File System Access API（新兴）
- OPFS（Origin Private File System）

## 三、方案横向对比

| 维度           | Cookie | LocalStorage | SessionStorage | IndexedDB | Cache API |
| -------------- | ------ | ------------ | -------------- | --------- | --------- |
| 容量           | ~4KB   | ~5MB         | ~5MB           | 很大      | 很大      |
| 生命周期       | 可设置 | 永久         | 会话           | 永久      | 手动管理  |
| 是否随请求发送 | ✅      | ❌            | ❌              | ❌         | ❌         |
| 数据格式       | 字符串 | 字符串       | 字符串         | 结构化    | Response  |

## 四、实践中的关键问题

### 1. 安全性

- XSS 攻击与存储安全
- Token 存储方案选择（Cookie vs LocalStorage 之争）
- 敏感数据加密存储

### 2. 数据序列化与反序列化

- JSON.stringify / JSON.parse 的坑
- 特殊类型处理（Date、RegExp、undefined、循环引用）

### 3. 存储容量管理

- 容量检测与超限处理
- 浏览器存储淘汰策略（Storage Eviction）
- 存储配额查询（`navigator.storage.estimate()`）

### 4. 跨标签页通信与同步

- `storage` 事件监听
- BroadcastChannel
- 多标签页数据一致性问题
