# AI Agent 入门指南

## 一、什么是 Agent？

用最直白的话说：

> **Agent = LLM（大语言模型）+ 感知 + 记忆 + 规划 + 工具使用 + 行动**

普通的 ChatGPT 对话是「你问一句，它答一句」。而 **Agent** 是让 AI 能**自主地**拆解任务、制定计划、调用工具、观察结果、反复迭代，直到完成目标。

------

## 二、核心概念拆解

### 1. LLM（大语言模型）—— Agent 的「大脑」

- GPT-4、Claude、LLaMA、Qwen 等
- 负责理解、推理、生成文本

### 2. Prompt / 指令

- 告诉 Agent「你是谁、你能做什么、你该怎么做」
- 比如 System Prompt 里写：「你是一个数据分析助手，可以调用 Python 和 SQL」

### 3. 规划（Planning）

Agent 拿到一个复杂任务后，需要把它拆成多个步骤：

```
用户：帮我分析这份销售数据，找出下滑原因，生成报告

Agent 的规划：
  Step 1: 读取数据文件
  Step 2: 做数据清洗
  Step 3: 分时间维度分析趋势
  Step 4: 找出异常点
  Step 5: 生成可视化图表
  Step 6: 撰写分析报告
```

常见的规划方法：

| 方法                       | 说明                                     |
| -------------------------- | ---------------------------------------- |
| **ReAct**                  | Reasoning + Acting，边思考边行动，最经典 |
| **Chain-of-Thought (CoT)** | 链式思考，一步步推理                     |
| **Plan-and-Execute**       | 先制定完整计划，再逐步执行               |
| **Tree of Thoughts (ToT)** | 树状搜索多种可能的推理路径               |

### 4. 记忆（Memory）

```
┌─────────────────────────────────┐
│           记忆系统               │
├──────────────┬──────────────────┤
│  短期记忆     │  长期记忆         │
│  (对话上下文)  │  (向量数据库存储)  │
│  Context      │  VectorDB        │
│  Window       │  (如 ChromaDB,    │
│               │   Pinecone)      │
└──────────────┴──────────────────┘
```

- **短期记忆**：当前对话的上下文（受 token 限制）
- **长期记忆**：把重要信息存到外部数据库，需要时检索回来（和 RAG 密切相关）

### 5. 工具使用（Tool Use / Function Calling）

这是 Agent 和普通聊天 AI 最大的区别之一：

```
# Agent 可以调用的工具示例
tools = [
    "搜索引擎",      # 查最新信息
    "代码执行器",     # 运行 Python/SQL
    "文件读写",       # 读取/保存文件
    "API 调用",      # 调用外部服务
    "数据库查询",     # 查数据库
    "浏览器",        # 打开网页
]
```

### 6. 观察与反馈循环（Observation Loop）

```
思考 (Thought) → 行动 (Action) → 观察 (Observation) → 思考 → 行动 → ...
```

这就是著名的 **ReAct 循环**，举个例子：

```
Thought: 用户问北京今天天气，我需要搜索
Action:  调用天气API("北京")
Observation: 晴，25°C，东风3级
Thought: 我拿到结果了，可以回复用户
Action:  回复用户
```

## 三、几个你会经常遇到的相关概念

| 概念                             | 一句话解释                                                   |
| -------------------------------- | ------------------------------------------------------------ |
| **RAG** (检索增强生成)           | 先检索相关文档，再让 LLM 基于文档回答，减少幻觉              |
| **Multi-Agent**                  | 多个 Agent 协作，比如一个写代码、一个审代码、一个测试        |
| **MCP** (Model Context Protocol) | Anthropic 提出的标准化协议，让 Agent 统一地连接各种工具和数据源 |
| **Function Calling**             | LLM 输出结构化的函数调用请求（而非纯文本）                   |
| **Hallucination** (幻觉)         | LLM 编造不存在的信息                                         |
| **Prompt Engineering**           | 设计好的提示词来引导 LLM 行为                                |
| **Embedding**                    | 把文本转为向量，用于语义搜索                                 |
| **Agentic Workflow**             | 用 Agent 思路来设计的工作流                                  |

## 六、一个最小 Agent 的伪代码

帮你建立直觉：

```
def agent(user_query):
    memory = []  # 对话历史
    tools = {"search": search_web, "calculator": calc, "code": run_python}
    
    memory.append({"role": "user", "content": user_query})
    
    while True:
        # 1. LLM 思考下一步
        response = llm.chat(
            system="你是一个助手，可以调用工具。请用ReAct格式回答。",
            messages=memory
        )
        
        # 2. 如果 LLM 决定调用工具
        if response.has_tool_call:
            tool_name = response.tool_call.name
            tool_args = response.tool_call.arguments
            
            # 3. 执行工具，获得观察结果
            observation = tools[tool_name](**tool_args)
            memory.append({"role": "tool", "content": observation})
        
        # 4. 如果 LLM 决定直接回复用户
        else:
            return response.content  # 最终答案
```

## 总结一句话

> **Agent 的本质就是：让 LLM 从「被动回答」变成「主动完成任务」，通过循环地思考-行动-观察来解决复杂问题。**



好吧，让我们回到agent领域吧，我没太理解agent框架和规划方法这两者之间的区别点，我可以理解为前者是一个代码工程例如vue react，后者是一类设计模式类似mvm. MVVM吗？



**框架 = 你用什么工具库来搭建 Agent（工程选择）**
**规划方法 = Agent 拿到任务后用什么策略去思考和行动（思维模式）**

你的类比方向对了，只是规划方法不是在规定"代码怎么组织"，而是在规定"LLM 怎么思考"——本质上是 **prompt 工程 + 控制流的组合策略**。

## 用代码感受一下区别

**同一个框架（LangChain），不同规划方法：**

Python

```
# ============ ReAct 策略 ============
# 边想边做，循环直到结束

agent = create_react_agent(llm, tools, prompt)
# 内部循环：Think → Act → Observe → Think → Act → ...


# ============ Plan-and-Execute 策略 ============
# 先出完整计划，再逐步执行

planner = LLMPlanner(llm)       # 负责出计划
executor = LLMExecutor(llm)     # 负责执行每一步

plan = planner.plan(user_query)  # 先规划
for step in plan:
    executor.execute(step)       # 再执行
```
